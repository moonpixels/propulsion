# Pushback Rules

Use this reference when a review finding should be rejected instead of implemented.

Push back when a finding:

- contradicts the plan
- breaks existing behavior
- assumes context the reviewer did not have
- adds unneeded scope
- is disproved by code or tests

Push back with:

- the relevant code or test evidence
- the plan or PRD constraint
- the concrete reason the finding is wrong here

## Rules

- Use this reference only when the finding is `invalid`.
- Do not use this reference to skip triage or rerun checks.
